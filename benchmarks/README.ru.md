## 📊 Тестирование производительности (Бенчмарки)

[English](README.md) | [Русский](README.ru.md)

Движок содержит полноценный набор автоматизированных микробенчмарков, работающих на базе библиотеки **BenchmarkDotNet**. Эти тесты позволяют непрерывно отслеживать скорость рендеринга, профилировать аллокации памяти в куче и определять границы эффективности бинарной сериализации данных в сети.

### ⚠️ Условия для получения точных метрик

Чтобы исключить искажения, вызванные планировщиком потоков операционной системы, скачками частоты процессора или обходом оптимизаций JIT-компилятора, строго соблюдайте следующие правила изоляции окружения перед запуском тестов:
* **Закройте все тяжелые фоновые процессы** (Docker-демоны, среды разработки IDE, браузеры или активные базы данных).
* **Подключите устройство к стабильному источнику питания** (не запускайте тесты на ноутбуках в режиме энергосбережения или с динамическими профилями производительности).
* **Никогда не запускайте бенчмарки с подключенным отладчиком (Debug)** или через стандартную команду `dotnet run` без флагов оптимизации.

### 🚀 Запуск бенчмарков

Перейдите в корневую директорию проекта бенчмарков в консоли и запустите тестирование, принудительно указав высокооптимизированный профиль компиляции **Release**:

```bash
# Перейти в директорию проекта бенчмарков
cd benchmarks/PixelTerminalUI.Benchmarks

# Запустить изолированный целевой бенчмарк в конфигурации Release
dotnet run -c Release -- --filter *SerializationBenchmark*
```

### 🗃️ Анализ отчетов и артефактов

После завершения всех итераций выполнения BenchmarkDotNet автоматически экспортирует точные таблицы с метриками в локальную директорию `./BenchmarkDotNet.Artifacts/results/`. Сводные отчеты распределяются по нескольким популярным форматам, включая markdown-таблицы (`.md`), файлы значений, разделенных запятыми (`.csv`), и автономные веб-страницы для наглядной презентации (`.html`).
